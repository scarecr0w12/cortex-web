# Migration Guide

This guide covers upgrading between CortexPrism versions and migrating data between installations.

## Version Compatibility

| From | To | Migration Required |
|------|----|--------------------|
| 0.0.x | 0.1.x | Database migration (`cortex migrate`) |
| 0.1.x | 0.2.x | Config format update |
| 0.2.x | 1.0.0 | Full migration (see below) |
| 0.52.x | 0.53.0 | DB migrations 044–047 (users, teams, tokens, scoping) |

## Checking Your Version

```bash
cortex --version
```

## Upgrading CortexPrism

### Source Install

```bash
cd ~/cortex
git pull origin main
deno run --allow-all src/main.ts setup    # Runs migrations and dependency updates
```

### Docker Install

```bash
docker pull cortexprism/cortex:latest
docker stop cortex-server
docker rm cortex-server
# Re-create with your docker run command
```

## Database Migrations

CortexPrism uses SQLite with automatic migration support:

```bash
# Run all pending migrations (idempotent, safe to run multiple times)
cortex db migrate
```

### Manual Migration

If automatic migration fails, you can run migrations manually:

```bash
# Backup first
cp ~/.cortex/data/cortex.db ~/.cortex/data/cortex.db.backup

# Then re-run migrate
cortex db migrate
```

## Data Directory Migration

To move your CortexPrism data to a new location:

```bash
# Stop all cortex processes
cortex daemon stop
cortex server start -s

# Copy data
cp -r ~/.cortex /new/location/.cortex

# Set new data directory
export CORTEX_DATA_DIR=/new/location/.cortex

# Start using new location
cortex daemon start
```

## Config Migration

When config format changes between versions:

```bash
# Backup existing config
cp ~/.cortex/config.json ~/.cortex/config.json.backup

# Run setup wizard to regenerate
cortex setup

# Or manually migrate databases:
cortex migrate
```

## Plugin Migration

When upgrading CortexPrism, plugins may need updates:

```bash
# Check plugin compatibility
cortex plugins list

# Update all plugins to latest
cortex plugins update --all

# Update a specific plugin
cortex plugins update my-plugin
```

## Rolling Back

If an upgrade causes issues:

```bash
# Restore database backup
cp ~/.cortex/data/cortex.db.backup ~/.cortex/data/cortex.db

# Revert to previous version
git checkout <previous-tag>
deno run --allow-all src/main.ts setup

# Update plugins to latest compatible versions
cortex plugins update --all
```

## Migrating from Other Tools

### From OpenAI Agents SDK

CortexPrism supports importing configurations from the OpenAI Agents SDK:

```bash
cortex import json --config agent_config.json
```

### From Custom Tools

If you have an existing tool configuration, you can wrap it as a CortexPrism plugin using the plugin system.

## Pre-Migration Checklist

1. **Back up databases** — `cp ~/.cortex/data/*.db ~/backup/`
2. **Export config** — `cp ~/.cortex/config.json config.backup.json`
3. **Check disk space** — Ensure enough space for migration
4. **Review changelog** — Check breaking changes in the [Changelog](/changelog)

## Post-Migration Checklist

1. **Verify version** — `cortex --version`
2. **Test chat** — `cortex agent chat "Hello" --model claude-sonnet-4-5`
3. **Check plugins** — `cortex plugins list`
4. **Verify data** — `cortex memory search "test"` (if you had memory)
5. **Check logs** — Review Lens audit log for errors
6. **Set up users** — `cortex users list` and create users if using multi-user

## 0.53.0 Migration Notes

The 0.53.0 release introduces multi-user collaboration with significant database changes:

- **Migrations 044–047** run automatically on startup. They create `users`, `teams`, `team_memberships`, `user_tokens`, `agents`, `resource_shares`, `instance_identity`, and `federation_peers` tables.
- An auto-admin user is created on first run. Default credentials: `admin` / `admin` (change immediately).
- Existing agents from `config.json` are preserved as fallback — they are migrated to the `agents` DB table.
- Existing resource rows are backfilled with admin ownership.
- No manual migration steps are required — all migrations are idempotent and safe to re-run.
