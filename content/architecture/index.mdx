# Architecture Overview

CortexPrism is a single-process AI Agent Operating System written in TypeScript/Deno. It exposes a CLI, a REST API + WebSocket server, and a web UI. All state is persisted in SQLite databases using WAL mode via `@libsql/client`.

## High-Level Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                         CortexPrism                             │
│                                                                 │
│   CLI (cortex agent chat / sandbox run / server start / ...)       │
│          │                                                      │
│          ▼                                                      │
│   ┌─────────────────────────────────────────────┐              │
│   │              agent/loop.ts                  │              │
│   │  userMessage → [metacognition assess]       │              │
│   │  → [memory inject] → [MQM model select]     │              │
│   │  → LLM call → [tool parse] → [validator]    │              │
│   │  → [LLM supervisor] → [execute]             │              │
│   │  → [re-prompt loop] → response              │              │
│   │  → [adversarial critique] → [episodic]      │              │
│   │  → [reflection] → [skill extract]           │              │
│   └─────────────────────────────────────────────┘              │
│          │                                                      │
│   ┌──────┼──────────────────────────────────────┐              │
│   │      │         Subsystems                   │              │
│   │  memory/   tools/   sandbox/   security/    │              │
│   │  llm/      server/  scheduler/  plugins/    │              │
│   │  memori/   skills/  metacog/   qm/          │              │
│   └──────────────────────────────────────────────┘             │
│                                                                 │
│   SQLite databases (WAL mode)                                   │
│   cortex.db · memory.db · lens.db · vault.db · plugins.db       │
└─────────────────────────────────────────────────────────────────┘
```

## Core Components

| Component | Description |
|-----------|-------------|
| **Agent Loop** | 7-stage orchestrator with metacognition assessment, memory injection, MQM model selection, LLM call, tool execution, adversarial critique, and post-turn processing |
| **Metacognition** | Pre-execution task assessment with confidence scoring, decision classification (direct/ask_first/delegate/plan_with_rollback/parallelize), and low-confidence auto-escalation |
| **Memory System** | 5-tier memory with hybrid FTS5 keyword + vector embedding retrieval and exponential decay scoring |
| **LLM Layer** | 30 provider implementations with unified `LLMProvider` interface and CascadeRouter |
| **Tool System** | 60+ built-in tools with 5-format tool call parsing (XML + bare JSON fallback) and extensible plugin registry |
| **Security (Parallax)** | Multi-layer defense with policy validation gate, LLM supervisor review for sensitive data, encrypted vault, regex deny rules, and full audit logging |
| **Security Supervisor** | 3-layer access control: data classification → LLM review of agent intent → human approval gate with 1-hour session caching |
| **Sandbox** | Docker container sandbox (subprocess fallback) with resource limits, Deno Worker sandbox for plugins, and auto-fix loop |
| **Pipeline System** | 12-stage hook-based processing pipeline (pre/post-assess, pre/post-reason, pre/post-llm, pre/post-tool, pre/post-reflect, pre/post-output) with 13 built-in hooks |
| **Plugin System** | ESM, MCP, and WASM plugins with 3 trust levels, sandbox isolation, supply-chain verification (SHA-256, signatures, malware scanning), and `window.Cortex` UI panel API |
| **Daemon Supervisor** | Background process manager for validator, executor, and scheduler daemons with auto-restart |
| **HTTP Server** | Built-in web server with REST API, WebSocket streaming, and web UI dashboard |
| **Scheduler** | SQLite-persisted cron job scheduler with retry |
| **Model Quartermaster** | Adaptive 6-signal model selection engine with reinforcement learning and context fingerprinting |
| **AgentLint** | Automated agent auditing with 33+ checks across agent configs, tool definitions, plugin manifests, and system prompts |
| **Memori Checkpoints** | Persistent agent checkpointing with 6-dimension state capture (conversation, memory, tools, reasoning, workspace, metadata) for crash survival and session resumption |
| **Git Workspace** | Agent-powered git porcelain with auto-commit messages, diff analysis, and branch management |
| **GitHub Integration** | PR, issue, and repository management with agent-powered code review and triage |
| **Sub-Agents** | 13 typed child agents (Explorer, Coder, Researcher, Planner, Generalist, Security Auditor, Debugger, Architect, DevOps, Data Analyst, UI/UX Designer, Code Reviewer, Technical Writer) with parallel dispatch |
| **Trigger System** | File watchers, webhooks, and git hooks that launch agent tasks on external events |
| **Workflow Engine** | DSL-based workflow execution with steps, branches, parallel execution, and approval gates |
| **Communication Channels** | 9 channel adapters (Discord, Slack, Teams, Telegram, WhatsApp, Google Chat, Lark, Mattermost, Rocket.Chat) |
| **Remote Agents** | Distributed node execution via WebSocket hub with capability-based security tiers |
| **MCP Server** | Model Context Protocol server exposing Cortex tools and agents to MCP clients |
| **Update System** | Dual-mode updates (binary/source) with SHA-256 verification and rollback |
| **Observability** | Prometheus-compatible metrics registry and distributed tracing with OTLP export |
| **Desktop Automation** | Cross-platform GUI automation (screenshots, clicks, typing, clipboard) for Linux, macOS, Windows |
| **Evaluation Framework** | Agent benchmarking with 8 task categories, pattern scoring, and regression detection |
| **Hub Node System** | Distributed hub with WebSocket-based node registry, capability tiers, and session routing |
| **Lens Audit Log** | Immutable event log for all system actions — LLM calls, tool execution, policy decisions |
| **Skills System** | Self-learning procedural knowledge extraction with 6-state lifecycle, embedding-based retrieval, and 4-tier trust system |
| **Multi-Agent Orchestration** | 6 strategies (sequential, parallel, debate, review-loop, hierarchical, graph DAG) for coordinating multiple agents |
| **Swarm Orchestration** | A2A-based distributed node clustering with coordinator, node registry, heartbeat monitoring, and resource snapshots |
| **Code Intelligence (Codegraph)** | Tree-sitter WASM parsing for 14+ languages with call graph resolution, impact analysis, and D3.js visualization |
| **Voice Pipeline** | Speech-to-text (Whisper), text-to-speech (OpenAI TTS / ElevenLabs), energy-based VAD, WebSocket streaming |
| **Computer Use** | GUI automation via Xvfb virtual displays with mouse, keyboard, and screenshot actions |
| **A2A Protocol Bridge** | Google Agent2Agent v1.0 with JSON-RPC 2.0, SSE streaming, server/client modes for cross-framework agent collaboration |
| **HEXACO Personality** | 6-dimension personality model driving prompt injection, memory retrieval bias, response style, and MQM routing hints |

## Data Flow

1. User input enters through CLI, Web UI, or API
2. Agent loop processes input with context from memory
3. LLM provider generates response via the model router
4. Tool calls are executed through the Parallax security gate
5. Results are stored in memory and returned to the user
